# Daily Report 日报自动生成

## 概述
自动生成工作日报并追加到当前工作目录下的 `日报.md`。自动获取日期、地点、天气，梳理当前对话中的工作内容，支持通过命令参数补充额外信息。

## 触发方式
- `/daily-report` — 自动梳理今日对话生成日报
- `/daily-report <任意补充内容>` — 梳理对话 + 补充内容追加到完成项

## 核心约束（必须遵守）

1. **追加而非覆盖**：只能在文档末尾新增内容，或更新已有日期的日报内容，不得整体覆盖文件
2. **不编造内容**：如果对话中信息不足以提炼出完整的日报内容，必须主动向用户提问获取缺失信息
3. **归档通知**：每次写入完成后，告知用户"日报已成功保存到 当前工作目录下的`日报.md`"
4. **保持格式一致**：新增内容必须与原有格式完全一致
5. **日期推算**：每次实时获取最新日期，正确推算"今天""昨天"等相对时间
6. **更新/追加逻辑**：如果指定日期的日报已存在，更新其内容；不存在则追加新内容
7. **工作总结同步更新**：新增完成项后，同步更新"今日工作总结"
8. **日期顺序检查**：变更后检查日报日期顺序，如果不是从小到大（升序），重新调整
9. **写入后验证**：文件操作完成后，立即检查文件内容是否完整，确认历史记录保留，核对新增内容正确写入

## 执行流程

### Step 1: 获取基础信息
- 获取当天日期，格式：`YYYY年MM月DD日`（如 `2026年05月29日`）
- 通过 IP 定位获取地点（如 `curl -s http://ip-api.com/json/?lang=zh-CN` 或 WebFetch 调用），提取城市
- 通过 WebSearch 查询当地当天天气（如 `深圳 今天 天气`）

### Step 2: 读取现有日报
- 读取 `当前工作目录下的`日报.md``
- 分析已有格式，确认最新一条日报的日期和结构
- 检查是否存在今天日期的日报条目

### Step 3: 梳理工作内容
- 回顾当前会话上下文，提取今日完成的工作事项
- 如果命令带有补充参数（如 `/daily-report 今日创建了日报总结技能`），将补充内容追加到完成项

### Step 4: 补充提问（如信息不足）
- 完成项不清晰 → 问："今天完成了哪些工作？"
- 代办项不明确 → 问："还有哪些未完成的事项？"
- 今日总结需要补充 → 问："有什么想补充到今日总结的吗？"
- 一次只问一个问题

### Step 5: 组装日报内容
严格按照现有日报格式组装。例如：

```
## 2026年05月29日

**地点**：深圳市

**天气**：深圳市，多云，气温 26°C ~ 31°C，北风 1-3 级

---

### 完成项

1. **项目名**
   - 具体内容

---

### 代办项

- 无

---

### 今日工作总结

总结内容

---
```

### Step 6: 写入文件
- 如果今天的日报不存在 → 在文档末尾追加新条目
- 如果今天的日报已存在 → 更新已有条目（追加完成项、更新总结）

### Step 7: 日期排序检查
- 读取文件中所有 `## YYYY年MM月DD日` 标题
- 检查是否为升序排列
- 如果不是，重新调整日期条目顺序

### Step 8: 写入后验证
- 重新读取文件，确认内容完整
- 确认历史记录未被覆盖或删除
- 确认新增/修改内容正确写入

### Step 9: 通知用户
- 告知用户"日报已成功保存到 当前工作目录下的`日报.md`"

## 格式参考

文件路径：`{当前工作目录}/日报.md`（自动检测）

标题：`# 工作日报`
日期标题：`## YYYY年MM月DD日`
字段格式：
```
**地点**：xxx
**天气**：xxx
```
条目分隔：`---`
完成项：`### 完成项` → 编号列表 `1. **标题**` → `   - 细节`
代办项：`### 代办项` → 无则 `- 无`，有则编号列表
总结：`### 今日工作总结` → 段落文字
条目间分隔：`---`

## 常见错误
- 不要整体覆盖文件，只追加或更新指定日期内容
- 不要忘记更新"今日工作总结"当新增了完成项时
- 不要忘记检查日期顺序
- 不要忘记写入后验证文件完整性
