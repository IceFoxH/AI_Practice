# Codex 高效工作流提示词手册

## 📋 各阶段提示词总结

### 阶段 1：项目初识

**场景**：首次接触项目，需要快速了解项目结构

**提示词**：
```
先不要修改代码。请先阅读这个项目，告诉我：
1. 项目结构是什么
2. 主要入口在哪里
3. 训练/测试/运行命令分别是什么
4. 哪些文件最可能和当前任务有关
5. 你不确定的地方有哪些
```

---

### 阶段 2：AGENTS.md 规则编写

**场景**：定义长期工作规则和实验原则

**提示词**：
```markdown
## 工作规则
- 默认先阅读相关代码和文档，再开始修改。
- 修改前先说明影响范围。
- 只做和当前任务相关的最小改动。
- 提交前必须运行最小验证。
- 说明用中文，代码、命令、文件名保持英文。
- 不要修改无关文件。

## 实验原则
实验必须服务于明确假设或决策。
不要为了补齐表格而穷举低价值 ablation。
```

---

### 阶段 3：复杂任务计划

**场景**：处理复杂任务，需要先设计方案

**提示词**：
```
先不要写代码。请进入计划模式，帮我设计实现方案：
1. 明确这次要解决的问题
2. 列出会影响哪些模块
3. 拆成 3-7 个步骤
4. 每一步怎么验证
5. 哪些地方最容易出错
等我确认后再开始实现
```

---

### 阶段 4：研究任务查证

**场景**：需要阅读论文或官方 repo 进行研究

**提示词**：
```
请先阅读论文和官方 repo，不要开始写代码。
先输出：
1. 论文核心 idea
2. 关键公式/模块
3. 官方实现和论文描述是否一致
4. 复现最小路径
5. 可能复现不出来的风险点
```

---

### 阶段 5：任务完成总结

**场景**：任务完成后进行状态总结

**提示词**：
```
请用一句话总结刚完成的任务、修改过的文件、当前状态和下一步注意事项。
```

---

### 阶段 6：验证完成状态

**场景**：确认任务完成质量，列出待确认项

**提示词**：
```
请最后给我：
1. 修改了哪些文件
2. 每个文件为什么改
3. 运行了哪些验证
4. 还有哪些没验证
5. 哪些地方需要我人工确认
```

## 🎯 完整工作流流程图

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   1. 项目初识    │ → │  2. AGENTS.md   │ → │  3. 复杂任务计划 │
│   (阅读项目)     │    │   (长期规则)     │    │   (设计方案)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                                       ↓
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  6. 验证完成     │ ← │  5. 任务完成     │ ← │  4. 研究任务     │
│  (确认状态)      │    │  (总结状态)      │    │   (查证对齐)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ↓
    /new 开新会话
```

## 💡 核心要点速查表

| 要点 | 说明 |
|:---|:---|
| **先阅读，后编码** | 避免带着错误假设开干 |
| **规则前置** | AGENTS.md 定义长期行为准则 |
| **计划先行** | 复杂任务必须拆解为 3-7 步 |
| **查证对齐** | 研究类任务先理解再动手 |
| **会话隔离** | 任务完成后用 `/new` 开启新会话 |
| **验证闭环** | 不要完全相信"完成了"，要求详细输出 |

| **阶段 6** | 验证确认 | 列出修改、验证、待确认项 |
