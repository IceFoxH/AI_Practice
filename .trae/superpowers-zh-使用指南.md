## 四、使用建议

### 最佳实践

1. **设计先于编码**
   - 收到功能需求时，先等待 brainstorming skill 自动触发
   - 明确需求后再动手写代码

2. **测试先于实现**
   - 写代码前先写测试（TDD）
   - 确保代码可测试、可验证

3. **验证先于完成**
   - 声称完成前必须运行验证命令
   - 用证据支撑断言

4. **善用中文 skills**
   - 代码审查时用 `/chinese-code-review`
   - 写文档时用 `/chinese-documentation`
   - 配置 Git 时用 `/chinese-git-workflow`

### 常见误区

❌ **误区一**：手动调用自动触发的 skills
- 自动 skills 会在合适时机自动触发，无需手动调用

❌ **误区二**：忽略手动调用的 skills
- 中文 skills 需要显式调用，不会自动触发

❌ **误区三**：跳过验证直接声称完成
- 必须运行验证命令，确认通过后才算完成
