   - **推荐工作流**：读上下文 → 写计划 → 确认范围 → 小步实现 → 跑验证 → 总结结果 → /new
   - **六大核心原则**：

     6. **不要完全相信"完成了"**：要求输出修改文件、修改原因、验证内容、未验证项、人工确认点

4. **Claude Code 扩展功能加载机制学习**
   - **核心扩展功能加载时机与内容总结表**：

| 功能 | 加载时机 | 加载内容 | 上下文成本 |
|------|----------|----------|------------|
| **CLAUDE.md** | 会话开始 | 所有 CLAUDE.md 文件的完整内容（托管、用户和项目级别） | 每个请求 |
| **Skills** | 会话开始 + 使用时（默认）；仅用户 skills 在调用时加载 | 启动时的描述，使用时的完整内容 | 低（每个请求的描述） |
| **MCP 服务器** | 会话开始 | 工具名称；完整架构按需加载 | 低，直到使用工具 |
| **Subagents** | 生成时 | 具有指定 skills 的新鲜上下文 | 与主会话隔离 |
| **Hooks** | 触发时（如 PostToolUse、SessionStart） | 无（外部运行），除非 hook 返回额外上下文 | 零，除非 hook 返回输出 |

   - **关键说明**：
     - CLAUDE.md 从工作目录读取直到根目录，并在访问子目录时发现嵌套文件
     - Skills 默认在会话开始时加载描述，完整内容在使用时加载；可通过 `disable-model-invocation: true` 完全隐藏直到手动调用
     - Subagents 拥有独立的上下文窗口，仅将结果摘要返回给主会话
     - Hooks 在生命周期事件上触发，执行 Shell 命令、HTTP 请求、LLM 提示或 subagent

   - **上下文窗口组成**：包含指令、读取的文件、响应以及后台加载的内容

     - 输入前：CLAUDE.md、自动内存、MCP工具名称、技能描述自动加载
     - 工作中：每个文件读取都会添加到上下文，路径范围规则自动加载
     - 后续：子代理在独立上下文窗口处理，仅返回摘要和元数据
   - **压缩后保留内容**：

     | 自动内存 | 从磁盘重新注入 |
     | 路径范围规则 | 丢失，需重新读取触发文件 |
     | 嵌套 CLAUDE.md | 丢失，需重新读取子目录文件 |
     | 技能主体 | 重新注入（每技能上限5000令牌，总计25000） |
   - **检查工具**：`/context` 查看实时上下文使用，`/memory` 检查加载的文件
